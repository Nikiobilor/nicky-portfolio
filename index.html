AWS EKS Terraform ArgoCD

Spring Petclinic — Microservices on EKS

Deployed the Spring Petclinic microservices suite to AWS EKS, provisioning the cluster, networking, and IAM access entries with Terraform instead of console clicks.

The real work was in the failures: EKS version mismatches that broke kubectl auth, IRSA permissions silently blocking the AWS Load Balancer Controller from issuing certificates, and account-level ALB restrictions that forced a pivot to NodePort and port-forwarding to keep services reachable.

GitHub Actions now handles image builds and tag pushes end-to-end, so a merge to main is the only manual step between a code change and a running pod.

About me

A bit about
who I am

I'm Nicky — a Cloud Engineer based in Lagos, Nigeria, with about two years of hands-on AWS experience. I hold the AWS Solutions Architect Professional, Terraform Associate, Linux Foundation Certified Practitioner, and CCNA certifications, and I spend most of my working hours in Terraform, Kubernetes, and Linux.

I learn by asking "why" before I touch syntax, and I'd rather rebuild a broken environment from scratch than patch around a problem I don't fully understand yet. Outside of infrastructure work, I run VOH Academy, designing curriculum and hands-on labs that take complete beginners to job-ready cloud engineers.
